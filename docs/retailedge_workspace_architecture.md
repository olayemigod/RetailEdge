# RetailEdge Workspace Architecture

RetailEdge uses the standard Frappe Workspace and Workspace Sidebar JSON assets synced by `retailedge.workspace_sync.sync_retailedge_workspace_layout` during migrate. The live workspace should expose only existing, safe links. Planned tools are documented here instead of being added as broken shortcuts.

## Link Types

- ERPNext Link: native ERPNext/POSNext source documents or reports that RetailEdge links to without duplicating. Examples: Bank Transaction, Payment Entry, Sales Invoice, Stock Entry, Stock Reconciliation, Item, Warehouse, POS Opening Shift, POS Closing Shift, Stock Ledger, and Stock Balance.
- RetailEdge Native: RetailEdge-owned documents and reports such as Cashier Expense, Daily Sales Audit, Payment Statement Import, Branch Performance Summary, Bank Transaction Matching, Unmatched Bank Transactions, Unmatched Bank Payment Events, Invoice Payment Audit, and Cashier Expense Review.
- RetailEdge Overlay: RetailEdge control/review layers that sit above native ERPNext evidence, such as Bank Match Review, Reconciliation Readiness, and Reconciliation Handoff.

## Live Workspace Sections

1. Operations
2. Review & Approvals
3. Reports & Analytics
4. Accounting / Ledger Bridge
5. Setup / Configuration
6. Admin / Maintenance

## Guardrails

RetailEdge must not duplicate ERPNext source-of-truth modules. Bank Transaction, stock ledger documents, accounting entries, Sales Invoice, and Payment Entry remain ERPNext/POSNext sources. RetailEdge should expose intelligence, review queues, operational controls, and safety overlays only where implemented.

## Home Workspace UX

The RetailEdge Home workspace is a compact operational launchpad. The sidebar keeps the six functional sections, while the Home page uses a dense, sequential shortcut grid so role and permission filtering does not leave scattered section blocks or empty headings.

### Dense Ordering Model

Home shortcuts are generated by `retailedge.workspace_home.HOME_WORKSPACE_ITEMS` and synced by `retailedge.workspace_sync.sync_retailedge_workspace_layout`. Each item records its label, target type, target name, section, priority, intended audience, and source classification. The sync path still reads the standard Workspace and Workspace Sidebar JSON assets, then builds the Home `content` and `shortcuts` from the registry with target existence checks.

When adding future Home links:

- Add the implemented target to the workspace JSON links only after the DocType, Report, Page, or safe ERPNext target exists.
- Add the Home shortcut to `HOME_WORKSPACE_ITEMS` with a clear section and priority.
- Keep daily operations first, reviews next, reports next, then accounting bridge, setup, and admin.
- Do not duplicate a shortcut on the Home page unless there is a clear operational reason.
- Do not expose roadmap placeholders as clickable cards.

### Visual Rules

RetailEdge Home cards use `retailedge/public/css/retailedge_workspace_home.css`. The stylesheet is scoped to the RetailEdge workspace route and should not style ERPNext, POSNext, or other workspaces. Keep the design plain: light cards, soft border, small shadow, compact spacing, and subtle ProcessEdge/RetailEdge accent only.

### Not Changed

The Home UX polish does not change business logic, accounting, reconciliation, stock, bank matching, expense, report logic, APIs, DocTypes, or permissions. Frappe permissions remain responsible for hiding links a user cannot access.

## Corrective Home Layout Rule

Frappe renders each `shortcut` content block as an independent EditorJS block with its own Bootstrap column class. If a shortcut is hidden by runtime permissions, the surrounding block can still affect layout, which creates visible gaps or scattered cards. RetailEdge Home therefore uses this safer pattern:

- One header block for light RetailEdge/ProcessEdge branding.
- One first shortcut block for POSNext `Start POS`, using POSNext's existing `/pos/` URL pattern.
- One native `card` block per non-empty section. Each card is backed by generated `Card Break` + child `Link` rows in `Workspace.links`.
- The card blocks restore section grouping while keeping each section's accessible child links compact inside the native Frappe links widget.

Future Home links should be added to `HOME_WORKSPACE_ITEMS`. URL launch actions should remain shortcuts unless Frappe's card/link widget gains explicit URL support. DocType, Report, Page, and Workspace items may be grouped inside section cards.
